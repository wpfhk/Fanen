## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 1,423만 개인투자자의 76.3%가 노후 준비 부족을 체감하고, 국민연금 소득대체율은 22%에 불과하다. 글로벌 정세가 투자에 미치는 영향은 커지지만, 일반 투자자는 "뉴스 → 수혜 섹터 → 연관 기업"으로 이어지는 Value Chain을 파악할 수단이 없다. |
| **Solution** | AI 어시스턴트 "반디(Bandi)"가 글로벌 정세를 감지하여 수혜 섹터/기업을 Value Chain(Tier 1~3)으로 자동 발굴하고, 비나 맵으로 시각화하며, 불로소득/배당 목표 달성을 설계해주는 무료 투자 인사이트 플랫폼. |
| **Target User** | 전 연령 불로소득/노후 대비 수요자 — 20대(자산 형성), 40대(은퇴 설계 시작), 60대(배당 현금흐름) |
| **Core Value** | "반디가 찾은 오늘의 기회, 당신의 내일이 빛나도록" — 정세 감지 + Value Chain 발굴 + 불로소득 설계의 삼위일체 |
| **Revenue Model** | 완전 무료 (사용자 확보 우선) → 추후 프리미엄 데이터/API B2B, 증권사 제휴 CPA |

# Part 2: Strategy Analysis

## 2. Value Proposition -- JTBD 6-Part Framework

### 2.1 Job Statement

| Part | 내용 |
|------|------|
| **When** | 글로벌 뉴스에서 지정학적 이벤트를 접했을 때 / 월 불로소득 목표를 설계할 때 / 배당주 포트폴리오를 구성할 때 |
| **I want to** | 그 이벤트가 어떤 산업과 기업에 기회를 만드는지 Tier 1~3까지 파악하고, 내 불로소득 목표에 맞는 포트폴리오를 설계하고 싶다 |
| **So I can** | 남들보다 먼저 숨은 수혜주를 발견하고, 안정적인 불로소득 파이프라인을 구축하여 노후를 대비할 수 있다 |
| **Functional Job** | 비나 맵에서 정세 이벤트를 클릭하면 수혜 섹터 → Value Chain(Tier 1~3) → 개별 종목까지 연결되고, 불로소득 계산기로 목표 달성 경로를 시뮬레이션한다 |
| **Emotional Job** | "나도 기관투자자처럼 정세를 읽고 숨은 기회를 찾을 수 있다"는 자신감, "노후 준비를 체계적으로 하고 있다"는 안도감 |
| **Social Job** | "이런 숨은 수혜주 어떻게 찾았어?"라는 주변의 반응, 투자 대화에서 Value Chain으로 분석하는 체계적 투자자 이미지 |

### 2.2 Gains & Pains

| Pains (해소해야 할 고통) | Gains (제공해야 할 이득) |
|------------------------|------------------------|
| 글로벌 뉴스를 봐도 수혜 종목까지 연결하지 못함 | 정세 이벤트 → 수혜 섹터 → Tier 1~3 기업까지 원클릭 체인 |
| 메이저 종목만 알지 숨은 수혜주(Tier 2~3)를 모름 | AI가 부품/소재/MRO/물류 등 간접 수혜 기업까지 자동 발굴 |
| "노후 준비 안 되어 있다" 76.3%의 불안감 | 불로소득 목표 계산기로 구체적 달성 경로 시각화 |
| 차트를 봐도 진입 타이밍을 모르겠음 | 반디가 신호등 + 쉬운 말로 "지금 이 가격대가 좋아요!" 안내 |
| 무료 투자 정보 앱이 없거나 기능 제한적 | 완전 무료로 모든 기능 이용 (사용자 확보 최우선) |
| 정보 과잉으로 판단 마비 | 반디의 모닝 라이트가 매일 3문장으로 핵심만 요약 |

| **5. Channels** | (1) 네이버/유튜브 SEO ("불로소득 계산", "월배당 ETF", "방산 수혜주") (2) 네이버 카페/투자 커뮤니티 (Value Chain 분석 콘텐츠 마케팅) (3) 카카오톡 공유 (비나 맵 스크린샷 바이럴) (4) 금융 유튜버 협업 |
| **6. Revenue Streams** | Phase 1: 완전 무료 (사용자 확보), Phase 2: (1) 증권사 제휴 CPA (계좌 개설 연동) (2) B2B 데이터 API (Value Chain 분석 데이터) (3) 프리미엄 실시간 알림 구독 |
| **7. Cost Structure** | (1) AI API (Gemini, FinBERT) — 사용량 비례 (2) 인프라 (Vercel, Railway 서울, Supabase, Upstash) — 월 고정 (3) KRX/DART/글로벌 뉴스 데이터 비용 (4) 인건비 |

| **9. Unfair Advantage** | (1) 한국어 금융 특화 AI (FinBERT+Gemini) (2) KRX/DART 공식 데이터 바인딩 (환각 방지) (3) Value Chain Tier 1~3 자동 발굴 (국내 유일) (4) 완전 무료 + 친근한 캐릭터 (진입장벽 제거) (5) 기존 Sprint 1~9 코드베이스 (빠른 MVP) |

## 12. Growth Loops

```
사용자가 비나 맵에서 기회 발견
→ Value Chain 분석 결과를 카카오톡/카페에 공유

```

```
→ 사용자가 관심 이벤트 클릭

→ "이걸 어떻게 찾았어?" 주변에 구전
→ 신규 사용자 유입
```

### 12.3 Portfolio Loop
```
불로소득 계산기로 목표 설정
→ 배당 포트폴리오 추천
→ 실제 증권사에서 매수 (CPA 수익)
→ 배당 수령 → "진짜 되네!" 확인
→ 목표 상향 → 추가 종목 탐색
→ 반복
```

### 14.2 Top 3 리스크 요약

1. **AI 정확도 리스크**: Value Chain의 Tier 2~3 기업 매핑이 부정확할 경우 서비스 신뢰도 치명타. 수동 검증 + 사용자 피드백 루프가 필수.
2. **수익화 타이밍 리스크**: 무료 전략은 사용자 확보에 유리하나, CPA/B2B 수익 없이 12개월 이상 운영 시 자원 고갈. 6개월 시점에 수익화 마일스톤 점검 필수.
3. **규제 리스크**: Value Chain "수혜주 발굴"이 "투자 자문"으로 해석될 가능성. "정보 서비스" 포지셔닝 + 면책 고지 + 법률 자문 필요.
